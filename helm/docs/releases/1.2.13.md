## [1.2.13](https://github.com/Cloudzero/cloudzero-agent/compare/v1.2.12...v1.2.13) (2026-07-01)

Release 1.2.13 extends the environment-variable injection introduced in 1.2.12 with per-component overrides, so individual components can set, override, or remove environment variables independently of the chart-wide defaults.

### Key Features

- **Per-Component Environment Variables**: Every component now accepts an `env` list that is layered on top of `defaults.env`. This is the supported way to set a variable that is only correct for one component — for example a `GOMAXPROCS` or `GOMEMLIMIT` tuned to a single container's CPU/memory limits, which would be a poor chart-wide default. The new paths are `components.agent.env`, `components.agent.reloader.env`, `components.validator.env`, `components.aggregator.collector.env`, `components.aggregator.shipper.env`, `components.webhookServer.env`, `components.webhookServer.backfill.env`, and `components.miscellaneous.{configLoader,helmless,initCert}.env`.

  Precedence (lowest to highest): `defaults.env` → `components.<component>.env` → chart-emitted lifecycle env → chart-emitted load-bearing literals (`SERVER_PORT`, `NODE_NAME`, `HOSTNAME`). So a component's `env` overrides `defaults.env` on a name collision but cannot override the chart's load-bearing variables. The deprecated top-level `server.env` continues to work as an alias for `components.agent.env`.

  A component entry with an explicit `value: null` acts as a **tombstone**: it removes a variable the component would otherwise inherit from `defaults.env`. (It cannot remove a chart-emitted variable — a higher-priority source re-sets it.)

  ```yaml
  defaults:
    env:
      - name: GOMAXPROCS
        value: "2"        # chart-wide default
  components:
    agent:
      env:
        - name: GOMAXPROCS
          value: "4"      # the agent overrides the shared default
    aggregator:
      shipper:
        env:
          - name: GOMAXPROCS
            value: null   # the shipper opts out entirely
  ```

### Upgrade Steps

To upgrade to version 1.2.13, run the following command:

```sh
helm upgrade --install <RELEASE_NAME> cloudzero/cloudzero-agent -n <NAMESPACE> --create-namespace -f configuration.example.yaml --version 1.2.13
```

No configuration changes are required. All `components.<component>.env` values default to empty, and the default install is unchanged.
